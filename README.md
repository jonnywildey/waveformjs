# Alphabets Heaven — waveformjs

Static music site featuring a spiral vinyl player. No build step, no bundler. Serve with any static server, e.g. `npx serve .`

## Prerequisites (for adding releases)

- **Node.js 18+** — runs `generate-batch.js`
- **ffmpeg** — audio decoding (`brew install ffmpeg`)

---

## Directory layout

```
waveformjs/
├── index.html                  ← releases hub (landing page)
├── stories.html                ← Stories vinyl player
├── eternal-infraviolet.html    ← Eternal Infraviolet Sunset EP player
├── <slug>.html                 ← one player page per release, at root
├── template.html               ← copy this when adding a new release
│
├── generate-batch.js           ← CLI: generates svg/ + json/ from MP3s
├── generate.html               ← browser-based generator (single file, legacy)
├── player.js                   ← shared player — reads window.albumConfig
│
├── style/
│   ├── alphabets.css           ← site styles (colours via CSS custom properties)
│   └── waveformjs.css          ← SVG player styles (hardcoded; lives inside <object>)
│
├── audio/
│   └── Alphabets Heaven - Album Name/
│       ├── 01 - Alphabets Heaven - Track.mp3
│       └── track-artwork.jpg        ← optional per-track background image
│
├── images/
│   ├── <slug>.jpg              ← main album artwork (background + panel + vinyl)
│   └── <slug>/
│       └── track-name.jpg      ← optional per-track artwork
│
├── svg/
│   └── <slug>/                 ← generated by generate-batch.js
│       └── track-name.mp3.svg
│
└── json/
    └── <slug>/                 ← generated by generate-batch.js
        └── track-name.mp3.json
```

**`<slug>`** is a short kebab-case identifier for the release (e.g. `eternal-infraviolet`).
It appears in the HTML filename, `svg/<slug>/`, `json/<slug>/`, and as the `svgId` prefix in `albumConfig`.

## Adding a new release

### 1. Prepare files

Place audio and artwork in the right directories:

```
audio/
  Alphabets Heaven - My Album/
    01 - Alphabets Heaven - Track One.mp3
    02 - Alphabets Heaven - Track Two.mp3
    track-two@0.5x.jpg          ← optional, for a track-specific background

images/
  my-album.jpg                  ← main album artwork
```

### 2. Generate waveform assets

```bash
node generate-batch.js "audio/Alphabets Heaven - My Album" my-album
```

Reads every `.mp3` in the directory and writes:

```
svg/my-album/track-one.mp3.svg
svg/my-album/track-two.mp3.svg
json/my-album/track-one.mp3.json
json/my-album/track-two.mp3.json
```

**Filename → slug convention:**  
`02 - Alphabets Heaven - Track Two.mp3` → `track-two.mp3`  
(last segment after ` - `, lowercased, spaces replaced with `-`)

Processing takes a few seconds per track. Progress is printed to the terminal.

### 3. Create the HTML page

Copy `template.html` to `my-album.html`. Fill in the two clearly marked blocks at the top of `<head>`.

**Block 1 — Palette** (controls every colour on the page):

```html
<style>
    :root {
        --panel-rgb:  11, 37, 64;         /* R,G,B of the dark frosted panels    */
        --accent:     #eb4116;             /* play button, active track, sliders  */
        --accent-rgb: 235, 65, 22;         /* same colour as R,G,B                */
        --bg-overlay: rgba(0, 0, 0, 0.8);  /* overlay darkness on the background  */
    }
    .content-container {
        background: linear-gradient(var(--bg-overlay), var(--bg-overlay)),
                    url('images/my-album.jpg') no-repeat center / cover;
    }
</style>
```

> **Why inline?** `url()` values in CSS custom properties resolve relative to the *stylesheet* file, not the HTML file. Putting the background rule in the page's own `<style>` block avoids that ambiguity.

**Block 2 — albumConfig:**

```js
window.albumConfig = {
    artist:  "Alphabets Heaven",
    title:   "My Album",
    artwork: "images/my-album.jpg",
    links: [
        { name: "Bandcamp",    url: "https://alphabetsheaven.bandcamp.com/album/my-album" },
        { name: "Apple Music", url: "https://music.apple.com/..." },
        { name: "Spotify",     url: "https://open.spotify.com/..." },
        { name: "Tidal",       url: "https://tidal.com/..." },
    ],
    tracks: [
        {
            id:    "track-one",
            url:   "audio/Alphabets Heaven - My Album/01 - Alphabets Heaven - Track One.mp3",
            svgId: "my-album/track-one.mp3",
            title: "Track One",
        },
        {
            id:    "track-two",
            url:   "audio/Alphabets Heaven - My Album/02 - Alphabets Heaven - Track Two.mp3",
            svgId: "my-album/track-two.mp3",
            title: "Track Two",
            artwork: "audio/Alphabets Heaven - My Album/track-two@0.5x.jpg", // optional
        },
    ],
};
```

### 4. Update the releases page

Add a link to `my-album.html` in `releases.html`.

## Reference

### albumConfig fields

| Field | Required | Description |
|---|---|---|
| `artist` | yes | Shown in the player header and track list |
| `title` | yes | Album/EP name |
| `artwork` | yes | Path to the main album image — used as background, panel thumbnail, and vinyl sticker |
| `links` | no | `{ name, url }` streaming links shown under the artwork panel |
| `tracks[].id` | yes | Unique slug — used in `?id=` URL param to deep-link a track |
| `tracks[].url` | yes | Path to the MP3 |
| `tracks[].svgId` | yes | `<slug>/<generated-name>.mp3` — must match `generate-batch.js` output exactly |
| `tracks[].title` | yes | Display name |
| `tracks[].artwork` | no | Track-specific image — crossfades the background, panel, and vinyl when selected |

### Palette variables

| Variable | Format | Controls |
|---|---|---|
| `--panel-rgb` | `R, G, B` bare (no `rgb()`) | Frosted dark panels: track list, artwork column, circle backdrop |
| `--accent` | hex | Play button fill, active track colour, slider thumbs, SVG stylus |
| `--accent-rgb` | `R, G, B` bare | Same accent for `rgba()` tints at different opacities |
| `--bg-overlay` | `rgba(...)` | Darkness of the overlay on the background image |

`--panel-rgb` uses bare `R, G, B` so the same value can be used at multiple opacities:  
`rgba(var(--panel-rgb), 0.85)` and `rgba(var(--panel-rgb), 0.5)` etc.
