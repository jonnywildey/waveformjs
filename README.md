# Alphabets Heaven — waveformjs

Static music site featuring a spiral vinyl player. No build step, no bundler. Serve with any static server, e.g. `npx serve .`

## Prerequisites (for adding releases)

- **Node.js 18+** — runs `generate-batch.js`
- **ffmpeg** — audio decoding (`brew install ffmpeg`)

---

## Directory layout

```
waveformjs/
├── generate-batch.js           ← CLI: generates svg/ + json/ from MP3s (run from releases/)
├── add-vignette.js             ← CLI: adds a dark radial vignette to artwork for backgrounds
│
├── audio/
│   └── Alphabets Heaven - Album Name/
│       ├── 01 - Alphabets Heaven - Track.mp3
│       └── Album Name.jpg          ← source artwork (copy into releases/images/)
│
└── releases/                   ← everything served to the browser lives here
    ├── index.html              ← releases hub (landing page / grid of cards)
    ├── <slug>.html             ← one player page per release
    ├── player.js               ← shared player — reads window.albumConfig
    │
    ├── style/
    │   ├── alphabets.css       ← site styles (colours via CSS custom properties)
    │   └── waveformjs.css      ← SVG player styles
    │
    ├── images/
    │   └── <slug>/
    │       ├── <slug>.jpg          ← main album artwork (artwork panel + vinyl label)
    │       └── <slug>back.jpg      ← vignetted version used as page background
    │
    ├── svg/
    │   └── <slug>/             ← generated by generate-batch.js
    │       └── track-name.mp3.svg
    │
    └── json/
        └── <slug>/             ← generated by generate-batch.js
            └── track-name.mp3.json
```

**`<slug>`** is a short kebab-case identifier for the release (e.g. `eternal-infraviolet`).
It appears in the HTML filename, `svg/<slug>/`, `json/<slug>/`, and as the `svgId` prefix in `albumConfig`.

## Adding a new release

### 1. Prepare artwork

Copy the artwork into `releases/images/<slug>/`:

```
releases/images/my-album/my-album.jpg
```

Then generate the vignetted background version:

```bash
node add-vignette.js releases/images/my-album/my-album.jpg
# writes: releases/images/my-album/my-albumback.jpg
```

### 2. Generate waveform assets

**Important: run this from inside the `releases/` directory**, otherwise the output lands at the repo root instead of `releases/svg/` and `releases/json/`.

```bash
cd releases
node ../generate-batch.js "/absolute/path/to/audio/Alphabets Heaven - My Album" my-album
```

Reads every `.mp3` in the directory and writes:

```
releases/svg/my-album/track-one.mp3.svg
releases/svg/my-album/track-two.mp3.svg
releases/json/my-album/track-one.mp3.json
releases/json/my-album/track-two.mp3.json
```

**Filename → slug convention:**  
`02 - Alphabets Heaven - Track Two.mp3` → `track-two.mp3`  
(last segment after ` - `, lowercased, spaces replaced with `-`)

For files without the ` - ` prefix pattern (e.g. `Alphabets Heaven - Track Two.mp3`), the same rule applies — the last segment is used.

Processing takes a few seconds per track. Progress is printed to the terminal.

### 3. Create the HTML page

Copy an existing release page (e.g. `releases/jays-odyssey.html`) to `releases/my-album.html`. Fill in the two blocks at the top of `<head>`.

**Block 1 — Palette** (controls every colour on the page):

```html
<style>
    :root {
        --panel-rgb:  11, 37, 64;         /* R,G,B of the dark frosted panels    */
        --accent:     #eb4116;             /* play button, active track, sliders  */
        --accent-rgb: 235, 65, 22;         /* same colour as R,G,B                */
        --bg-overlay: rgba(0, 0, 0, 0.8);  /* overlay darkness on the background  */
    }
    .content-container {
        background: linear-gradient(var(--bg-overlay), var(--bg-overlay)),
                    url('images/my-album/my-albumback.jpg') no-repeat center / cover;
        background-attachment: fixed;
    }
</style>
```

> **Why inline?** `url()` values in CSS custom properties resolve relative to the *stylesheet* file, not the HTML file. Putting the background rule in the page's own `<style>` block avoids that ambiguity.

**Block 2 — albumConfig:**

```js
window.albumConfig = {
    artist:  "Alphabets Heaven",
    title:   "My Album",
    year:    2024,
    label:   "Label Name",       // or "" if self-released
    credits: [                   // or [] if none
        "Some Person (saxophone)",
    ],
    artwork: "images/my-album/my-album.jpg",
    links: [
        { name: "Bandcamp",    url: "https://alphabetsheaven.bandcamp.com/album/my-album" },
        { name: "Spotify",     url: "https://open.spotify.com/..." },
        { name: "Apple Music", url: "https://music.apple.com/..." },
        // Download link — rendered under a separate "Download" heading:
        { name: "MP3s (zip)", url: "https://pub-b2c73c4391b6462980c9e5d8c74b3b8f.r2.dev/audio/Alphabets Heaven - My Album.zip", download: true },
    ],
    tracks: [
        {
            id:    "track-one",
            url:   "https://pub-b2c73c4391b6462980c9e5d8c74b3b8f.r2.dev/audio/Alphabets Heaven - My Album/01 - Alphabets Heaven - Track One.mp3",
            svgId: "my-album/track-one.mp3",
            title: "Track One",
        },
        {
            id:    "track-two",
            url:   "https://pub-b2c73c4391b6462980c9e5d8c74b3b8f.r2.dev/audio/Alphabets Heaven - My Album/02 - Alphabets Heaven - Track Two.mp3",
            svgId: "my-album/track-two.mp3",
            title: "Track Two",
        },
    ],
};
```

> **Audio URLs:** audio is served from Cloudflare R2, not from the local `audio/` directory. The base URL is:  
> `https://pub-b2c73c4391b6462980c9e5d8c74b3b8f.r2.dev/audio/<folder>/<file>.mp3`  
> Spaces in folder/file names can be left as-is — browsers handle encoding automatically.

### 4. Update the releases hub

Add a card to `releases/index.html` inside the `.releases-grid` div:

```html
<a class="release-card" href="my-album.html">
    <div class="release-artwork">
        <img src="images/my-album/my-album.jpg" alt="My Album" />
    </div>
    <div class="release-info">
        <div class="release-title">My Album</div>
        <div class="release-label">Label Name · 2024</div>
    </div>
</a>
```

## Reference

### albumConfig fields

| Field | Required | Description |
|---|---|---|
| `artist` | yes | Shown in the player header and track list |
| `title` | yes | Album/EP name |
| `year` | no | Release year, shown in the player header |
| `label` | no | Label name, shown in the player header |
| `credits` | no | Array of strings shown below the artwork |
| `artwork` | yes | Path to the main album image — used as artwork panel and vinyl sticker |
| `links` | no | Array of link objects (see below) shown under the artwork panel |
| `tracks[].id` | yes | Unique slug — used in `?id=` URL param to deep-link a track |
| `tracks[].url` | yes | Full R2 URL to the MP3 |
| `tracks[].svgId` | yes | `<slug>/<generated-name>.mp3` — must match `generate-batch.js` output exactly |
| `tracks[].title` | yes | Display name |

### Link objects

| Field | Description |
|---|---|
| `name` | Link text |
| `url` | Destination URL |
| `download` | `true` to show under a "Download" heading instead of "Listen on" |

### Palette variables

| Variable | Format | Controls |
|---|---|---|
| `--panel-rgb` | `R, G, B` bare (no `rgb()`) | Frosted dark panels: track list, artwork column, circle backdrop |
| `--accent` | hex | Play button fill, active track colour, slider thumbs, SVG stylus |
| `--accent-rgb` | `R, G, B` bare | Same accent for `rgba()` tints at different opacities |
| `--bg-overlay` | `rgba(...)` | Darkness of the overlay on the background image |

`--panel-rgb` uses bare `R, G, B` so the same value can be used at multiple opacities:  
`rgba(var(--panel-rgb), 0.85)` and `rgba(var(--panel-rgb), 0.5)` etc.

### Tools

#### `generate-batch.js`

```
node generate-batch.js <audio-directory> <album-slug>
```

Must be run from inside `releases/` so output goes to `releases/svg/` and `releases/json/`.

#### `add-vignette.js`

```
node add-vignette.js <image.jpg> [image2.jpg ...]
```

Adds a dark radial vignette to each image and saves a `<name>back<.ext>` alongside it. Strength and inner radius are tunable at the top of the file.
